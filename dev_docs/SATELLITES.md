# Satellite Visualization — Architecture Notes

## Overview

BeltViz renders a curated set of real satellites propagated from TLE data using the US Space Force
SGP4 WASM propagator. Satellites appear as colored dots in the 3D scene, grouped by orbit class.
Selecting a satellite shows an animated orbit trace and displays its magnetic environment
(L-shell, |B|, region) in the existing environment readout overlay.

## Data Pipeline

### Input

`public/data/Space-Track.all.3le.txt` — 3LE (3-line element) format downloaded from
[space-track.org](https://www.space-track.org/). Each record is three lines: name, TLE line 1,
TLE line 2. Approximately 30,000 objects as of the Feb 2026 snapshot.

### Debris filtering

Objects whose name contains any of the following substrings (case-insensitive) are excluded:

```
DEB, R/B, DEBRIS, FRAG, ADAPTER, TANK, SHROUD, AKM, PLF, (YO), TBA -, ROCKET, OBJECT, PAYLOAD COVER
```

### Orbit classification (Space-Track official definitions)

Applied in priority order from TLE line 2 fields:

```
meanMotion   = parseFloat(line2.substring(52, 63))  // revolutions/day
eccentricity = parseFloat("0." + line2.substring(26, 33))
period       = 1440 / meanMotion                    // minutes

HEO:  eccentricity > 0.25
LEO:  meanMotion > 11.25 AND eccentricity < 0.25
GEO:  0.99 <= meanMotion <= 1.01 AND eccentricity < 0.01
MEO:  600 <= period <= 800 min AND eccentricity < 0.25
OTHER: everything else (sub-orbital, super-GEO, transitional, etc.)
```

### Notable set selection (default ~1,500)

1. **Priority 1 — named satellites**: Name substring match against a list including ISS, Hubble,
   JWST, Chandra, GOES, NOAA, NAVSTAR (GPS), GLONASS, GALILEO, BEIDOU, MMS 1-4, RBSP A/B,
   SWARM A/B/C, THEMIS A-E, ARASE, Sentinel, Landsat, Terra, Aqua, etc.
2. **Priority 2 — megaconstellations** (capped for performance): STARLINK (100), ONEWEB (30),
   IRIDIUM (25), ORBCOMM (10), GLOBALSTAR (10).
3. **Priority 3 — filler**: One satellite per unique name-prefix to fill the remaining budget.

### Output format (`public/data/satellites.json`)

```json
{
  "version": "1.0",
  "generated": "<ISO timestamp>",
  "count": 1500,
  "satellites": [
    {
      "id": 25544,
      "name": "ISS (ZARYA)",
      "orbitClass": "leo",
      "notable": true,
      "line1": "1 25544U ...",
      "line2": "2 25544  ..."
    }
  ]
}
```

Satellites are sorted by orbit class order (LEO, MEO, GEO, HEO, OTHER) then by NORAD ID.

## SGP4 WASM Propagator

### Source

US Space Force official SGP4 propagator, version 9.1.1.0. WASM build generated by Emscripten.
Source: `dev_docs/v9.1.1.0_C_Sgp4Prop_Wasm/`. Example usage in
`dev_docs/v9.1.1.0_C_Sgp4Prop_Wasm/example/Sgp4Prop.html`.

### Why a classic worker?

The Emscripten module uses `importScripts()` for its WASM loader, which is only available in
classic (non-module) web workers. The satellite worker is therefore placed in `public/workers/`
(served as a static asset, not bundled by Vite) and created with:

```js
new Worker('/workers/satelliteWorker.js')   // no { type: 'module' }
```

All other BeltViz workers use `{ type: 'module' }` and Vite bundling.

### WASM API

Wrapped via `Module.cwrap`:

| Function | Signature | Purpose |
|----------|-----------|---------|
| `TleAddSatFrLines_wasm` | `(line1, line2) → i64 satKey` | Parse one TLE, return opaque key (0 on failure) |
| `Sgp4InitSats_wasm` | `(satKeysPtr, count, errPtr) → code` | Initialize SGP4 structures (must call before propagation) |
| `Sgp4PropDs50UtcPosVel_wasm` | `(satKeysPtr, count, ds50utc, nSteps, stepMin, resultPtr) → code` | Propagate to absolute time; `nSteps=1, stepMin=1.0` for swarm ticks |
| `Sgp4RemoveSats_wasm` | `(satKeysPtr, count, errPtr) → code` | Free propagator memory |
| `TleRemoveSats_wasm` | `(satKeysPtr, count, errPtr) → code` | Free TLE memory |

### Result buffer layout

`Sgp4PropDs50UtcPosVel_wasm` writes **8 doubles per satellite per step** into the provided buffer:

```
[errCode, propTime_Ds50, x_km, y_km, z_km, vx_km_s, vy_km_s, vz_km_s]
```

`errCode = 0` means success. Non-zero means the satellite has decayed or the TLE is invalid.

### Time: Ds50UTC

Days since 1950-01-01 00:00:00 UTC:

```js
const DS50_OFFSET_MS = 631_152_000_000;  // ms from 1950-01-01 to Unix epoch
const ds50utc = (date.getTime() + DS50_OFFSET_MS) / 86_400_000;
```

## Coordinate System

SGP4 returns **ECI / TEME** coordinates (km, z-up). BeltViz uses **Y-up scene coordinates**
(unitless, Earth radius = 1.0). Conversion in the worker:

```
ECI (TEME, z-up, km) → ECEF (rotate around z by GMST) → BeltViz (Y-up, unitless)

x_bv = (x_eci * cos(gmst) + y_eci * sin(gmst)) / Re
y_bv = z_eci / Re
z_bv = (-x_eci * sin(gmst) + y_eci * cos(gmst)) / Re
```

where `Re = 6371.2 km` and GMST is Greenwich Mean Sidereal Time in radians.

**GMST formula** (Vallado, accurate to ~0.1 arcsec for modern dates):

```
JD_1950 = 2433281.5
jd0h    = floor(ds50utc) + JD_1950
T       = (jd0h - 2451545.0) / 36525.0       // Julian centuries from J2000
gmst_0h = 100.4606184 + T*(36000.77004 + T*(0.000387933 - T/38710000))  // degrees
gmst    = (gmst_0h + 360.98564724 * frac_day) % 360                     // degrees
```

## Propagation Worker Protocol

```
Main → Worker: { type: 'init', satellites: [{id, line1, line2}, ...] }
Worker → Main: { type: 'ready', count }

Main → Worker: { type: 'propagate', requestId, ds50utc }
Worker → Main: { type: 'positions', requestId, positions: Float32Array, count }
               (positions transferred zero-copy; layout: [x0,y0,z0, x1,y1,z1, ...] scene coords)

Main → Worker: { type: 'traceOrbit', requestId, satIndex, ds50utc, periodMin }
Worker → Main: { type: 'orbit', requestId, satIndex, points: Float32Array }
               (points transferred zero-copy; 181 × [x,y,z] scene coords = one full orbit)
```

**Stale-response guard**: `requestId` is an incrementing counter on the main thread. The worker
echoes it back; main.js discards any response whose `requestId` doesn't match the latest.

**Orbit trace**: 181 points propagated at `stepMin = periodMin / 180`. This produces 180 segments
that close the orbit. `periodMin = 1440 / meanMotion` (from TLE line 2).

## Rendering (`src/scene/satelliteSwarm.js`)

### Dot rendering

One `THREE.Points` per orbit class in a `THREE.Group`. Custom `ShaderMaterial`:

- Vertex attributes: `satColor` (vec3 rgb), `satSize` (float)
- Uniform: `uDPR` (device pixel ratio for size correction)
- Size attenuation: `clamp(satSize / (dist * 0.15 + 0.04), 2.0, 18.0) * uDPR`
- Fragment: radial alpha falloff `pow(1.0 - d, 1.8)`, discard outside unit circle
- `blending: THREE.AdditiveBlending`, `depthWrite: false`

**Orbit class colors:**

| Class | Color | Hex |
|-------|-------|-----|
| LEO | Pale blue-white | `#c8d8f0` |
| MEO | Cyan-teal (GPS/nav) | `#44eebb` |
| GEO | Gold (geostationary ring) | `#ffdd44` |
| HEO | Magenta (Molniya orbits) | `#ee66ff` |
| OTHER | Grey | `#888888` |

**Selected satellite**: rendered white at 2.5× base size.

### Orbit trace

`THREE.Line` + `LineBasicMaterial` (color matches orbit class). `geometry.setDrawRange(0, n)` grows
from 0 to 181 over 2000 ms using `performance.now()` timestamps. The trace holds at full length
while the satellite remains selected. Deselecting disposes geometry and material immediately.

## UI (`src/ui/satellitePanel.js`)

HTML overlay panel (`position: fixed; right: 310px; top: 60px`) adjacent to the lil-gui panel.

- Live search input with 200 ms debounce
- Scrollable results list (max 80 rows shown; prompts to refine if more)
- Orbit class color badges on each row
- Selected satellite info section: name, NORAD ID, orbit class, estimated altitude, period
- Closes on Escape or click outside (the "Search / Select Satellite" GUI button is exempt)

State helpers in `src/ui/satelliteSearchState.js`:

- `filterSatellites(satellites, query, orbitClasses, notableOnly)` — name/NORAD ID substring search
- `findSatelliteIndex(satellites, noradId)` — lookup by NORAD integer ID
- `periodFromLine2(line2)` — `1440 / meanMotion` in minutes

## URL Persistence

| Param | Type | Default | Meaning |
|-------|------|---------|---------|
| `satSwarm` | bool | false | Satellite swarm enabled |
| `satSelected` | int | (omitted) | NORAD ID of selected satellite |

Example: `#satSwarm=true&satSelected=25544` restores the ISS selected with swarm enabled.

On load, `readFromUrl()` returns `satellites.enabled` in the nested params object (merged via
`Object.assign(params.satellites, ...)`) and `_satSelected` as a transient top-level key.
After the satellite catalog loads, `findSatelliteIndex` resolves the NORAD ID to a catalog index.

## Propagation Cadence

| Simulation speed | Real-time interval between propagations |
|------------------|-----------------------------------------|
| 1× | 10,000 ms |
| 60× | 2,000 ms |
| 3,600× | 500 ms |
| 86,400× | 200 ms |

On date jump (pause/step/drag), `lastPropTime` is reset to 0 to force an immediate re-propagation.
